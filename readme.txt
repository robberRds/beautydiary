Назва: BeautyDiary

Коротко про додаток:
BeautyDiary — мобільний додаток для ведення записів клієнтів салону краси або приватного майстра. Дозволяє зберігати записи (дата/час, клієнт, послуга, ціна, телефон, нотатки), додавати фото для кожного запису, переглядати історію візитів, статистику та експортувати дані в PDF/Excel.

Для кого:
- Майстри салонів краси
- Власники невеликих студій / фрилансери
- Адміністратори, які хочуть мати простий та локальний облік записів

Основні можливості:
- Створення та редагування записів з полями: ім'я клієнта, дата і час, телефон, ціна, замітки, фото.
- Збереження фото з галереї (камеру прибрано — тільки вибір з галереї).
- Збереження шляху до фото в базі даних (`photoPath`) і відображення у звітах/експорті.
- Перегляд фото в повноекранному режимі з масштабуванням (pinch-to-zoom).
- Галерея фото з пошуком та посторінковою навігацією (пагінація).
- Історія візитів клієнтів з фільтрами та пагінацією.
- Календар з позначками днів ("трафік"): кольори залежно від кількості записів (зелений/помаранчевий/червоний).
- Налаштування нагадувань:
  - Щоденний ранковий огляд (morning summary) о вибраному часі.
  - Нагадування перед подією (pre-event) з налаштовуваним відступом (хвилини до заходу).
  - Загальні налаштування: вмикати/вимикати нагадування, години/хвилини ранкового повідомлення.
- Автоматичні/ручні резервні копії (backup) даних, з можливістю синхронізації при потребі.
- Експорт даних у PDF та Excel з українськими заголовками та колонкою "Фото" (шлях до зображення), форматування ціни з символом ₴.
- Статистика: групування даних, гістограми/зведення виторгу за періоди.
- Розширений ввід: relaxed Cyrillic input (покращений ввід кирилицею) та українська локалізація інтерфейсу.
- Просте керування часом у UI: за замовчуванням час нового запису — 00:00; поля дати і часу підкреслені та клікабельні; кнопка зміни часу виразна.
- Видалення підтверджень: можливість керувати підтвердженнями видалення (за вимогою змінено/видалено).

Технічні деталі:
- Платформа: Flutter (Dart)
- Локальна база: `sqflite` (SQLite)
- Пакети: `flutter_local_notifications`, `timezone`, `image_picker`, `pdf`, `printing`, `excel`, `intl`, `shared_preferences`, `path_provider` та ін.
- Архітектура: сервіси-синглтони для DB, Notifications та Export; екрани в `lib/screens`, сервіси в `lib/services`, моделі у `lib/models`.
- Міграція БД: додано поле `photoPath` у схемі для збереження шляхів до фото.

Запуск додатку (локально):
1. Встановіть Flutter та Android SDK.
2. У корені проєкту виконайте:

```bash
flutter pub get
flutter run -d <device-id або emulator-id>
```

3. На Android-емуляторі точність точних сигналів (exact alarms) може бути обмежена — для перевірки нагадувань раджу тестувати на реальному пристрої.

Структура проекту (коротко):
- `lib/main.dart` — точка входу.
- `lib/models/appointment.dart` — модель запису.
- `lib/services/db_service.dart` — доступ до БД і тригери (викликає `NotificationService().rescheduleAll()` при зміні записів).
- `lib/services/notification_service.dart` — ініціалізація та планування нагадувань (morning summary, pre-event), скасування.
- `lib/services/export_service.dart` — експорт у PDF/Excel.
- `lib/screens/*` — екрани: `new_appointment_screen.dart`, `photo_viewer_screen.dart`, `photo_gallery_screen.dart`, `home_screen.dart`, `visit_history_screen.dart` тощо.
- `pubspec.yaml` — залежності й ресурси (assets).

Відомі обмеження та рекомендації:
- Емулятори часто не підтримують exact alarms -> в логах можна бачити `exact_alarms_not_permitted`; додаток автоматично переходить на inexact режим у таких випадках.
- Для коректної роботи часових зон і щоб уникнути зсувів у нагадуваннях, тестуйте на реальних пристроях у вашій локальній часовій зоні.
- Деякі залежності мають новіші версії; при потребі можна оновити `pubspec.yaml` і перевірити сумісність.
